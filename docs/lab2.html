基于感知 - 通信 - 可视化的 C-V2X 系统构建

从真实场景感知数据生成到通信传输，再到状态可视化的全过程实验验证。

实验动机与目的

随着车路协同技术的发展，当前 C-V2X 研究存在两个关键痛点：一是缺少全过程实验验证平台，导致理论与实验脱节；二是抽象通信指标难以直接体现交通价值。

现有实验往往依赖纯仿真数据，缺少对真实交通场景中目标感知动态性与链路不确定性的还原。本实验旨在构建一套完整的全链路系统，填补这一空白。

  • 构建“感知→传输→可视化”全流程系统。
  • 基于真实 YOLOv5 视频数据还原动态特征。
  • 开发交互式 Dashboard 实现实时监控。
  • 验证协议与可视化模块的兼容性。

系统整体架构

本实验覆盖“数据生成 - 传输 - 监控”三个核心环节，形成了完整的闭环验证体系。

核心模块功能实现

系统通过以下三个核心模块的协同工作，将原始视频数据转化为可视化的通信态势。

1. 交通目标感知
  • 核心算法：使用 YOLOv5 对夜间交通视频 (traffic_night.mp4) 进行逐帧处理。
  • 数据提取：识别车辆、行人类别，提取 Bounding Box 与置信度。
  • 语义转换：将检测结果封装为带时间戳的 JSON 对象，作为通信载荷。
2. C-V2X 数据传输
  • 链路模拟：基于 Python Socket (TCP) 模拟“车载(OBU)-路侧(RSU)”链路。
  • 并发支持：RSU 监听端口，支持多 OBU 并发接入与数据解析。
  • 传输控制：通过 send_interval 控制频率，模拟真实协同传输。
3. 可视化应用
  • 实时监控：左侧显示 YOLO 检测画面，右侧显示通信日志。
  • 场景重绘：底部简易交通场景重绘（道路、车辆位置）。
  • 数据解析：汇总感知数据，为 Dashboard 提供输入。

V2X 通信状态可视化 Dashboard

为增强通信性能分析的直观性，我们设计了基于浏览器的交互式 Dashboard。它加载并解析本地 CSV 格式的 V2X 记录数据（源自真实 VANET 数据集子集，约 2000 条高质量记录）。

核心功能：

  • 多维指标监控：端到端时延 (Latency)、数据包投递率 (PDR)、吞吐量。
  • 真实数据驱动：保留了链路波动与丢包现象，还原真实通信不确定性。
  • 三级可视化结构：总体态势概览 + 细粒度时间趋势 + 原始事件明细。
  • 交互筛选：支持按车辆 ID 和时间轴过滤，快速定位异常。

实验意义与研究价值

本实验并非单纯的性能对比，而是作为一种系统理解工具，揭示了 V2X 通信在真实交通环境中对协同感知的基础性作用。

教学与展示价值

采用“数据子集 + 交互可视化”模式，提供了一种低门槛、高表达力的研究范式，适合快速理解全链路流程。

增强直观性

将抽象的通信指标转化为具有空间语义和时间语义的系统行为，使看不见的“信号”变成了看得见的“态势”。

闭环验证基础

为前文的仿真分析提供了现实场景支撑，完善了“机制差异—系统行为”的逻辑链条，为后续实时闭环系统奠定基础。
